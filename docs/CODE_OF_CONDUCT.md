# Code of Conduct

## Purpose

This Code of Conduct establishes expectations for respectful, professional, and responsible participation in this research software project.

It applies to repository discussions, issues, pull requests, reviews, correspondence, and other project-related spaces.

## Expected Behaviour

Participants are expected to:

- communicate respectfully and professionally;
- provide constructive, evidence-based feedback;
- respect differences in experience, discipline, culture, identity, and perspective;
- focus criticism on ideas, methods, code, and evidence rather than individuals;
- acknowledge contributions accurately;
- disclose material conflicts of interest;
- protect confidential, personal, and sensitive information;
- avoid sharing API keys, credentials, or restricted data;
- document methodological changes honestly;
- correct errors transparently;
- respect software, data, and publication licences.

## Unacceptable Behaviour

Unacceptable conduct includes:

- harassment, intimidation, discrimination, or personal attacks;
- insulting, demeaning, threatening, or sexually inappropriate communication;
- deliberate disruption of project discussions;
- publishing another person's private information without permission;
- misrepresenting results, contributions, affiliations, or approvals;
- fabricating, falsifying, or selectively concealing research outputs;
- removing required attribution;
- submitting code, data, or text without the right to do so;
- exposing credentials, confidential information, or identifiable participant data;
- pressuring contributors to conceal methodological limitations or errors;
- retaliating against a person who reports a concern in good faith.

## Research Integrity

Because this repository supports scientific work, participants must preserve:

- traceability of experimental configurations;
- integrity of datasets and labels;
- separation of original and re-run outputs;
- transparency about model substitutions;
- accurate reporting of failed calls, unknown predictions, and exclusions;
- consistency between the repository, manuscript, and archived release.

No contributor should silently modify archived outputs or present a new run as the original run.

## Scope

This Code applies whenever a person is representing the project or participating in a project-managed space, including:

- GitHub issues and pull requests;
- code review;
- project email;
- research meetings;
- release preparation;
- data and documentation exchanges.

## Reporting a Concern

Concerns should be reported privately to the repository maintainer or the appropriate project lead.

A report should include, where possible:

- what occurred;
- when and where it occurred;
- who was involved;
- relevant links or records;
- any immediate safety, privacy, or research-integrity risk.

Do not disclose sensitive details publicly when a private report is more appropriate.

## Response and Enforcement

Project maintainers may take proportionate action, including:

- requesting a correction or clarification;
- editing or removing inappropriate content;
- rejecting or closing a contribution;
- restricting participation;
- documenting a research-integrity concern;
- escalating the matter through the relevant institutional process.

Reports will be considered as confidentially as reasonably possible. Retaliation against a good-faith reporter is prohibited.

## Fair Process

Where appropriate, the person whose conduct is under review should be informed of the concern and given an opportunity to respond.

Decisions should consider:

- severity;
- intent;
- impact;
- recurrence;
- privacy and safety;
- research-integrity obligations;
- institutional requirements.

## Attribution and Updates

This project-specific Code of Conduct may be updated as the collaboration, publication, or institutional requirements evolve.

Material changes should be documented in [`CHANGELOG.md`](CHANGELOG.md).
